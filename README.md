# elevators_site

## Настройка переменных окружения

Для отправки заявок с формы проект использует серверный endpoint и почтовый провайдер `Resend`. Все секреты должны храниться только на сервере в файле `.env` и не должны попадать в клиентский код.

### 1. Создайте файл `.env`

В корне проекта создайте файл `.env`. Проще всего взять за основу файл [.env.example](c:/Users/сережка/Documents/Projects%20coding/elevators_site/.env.example) и скопировать его содержимое.

Пример:

```env
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxx
LEAD_TO_EMAIL=manager@example.ru
LEAD_FROM_EMAIL=site@example.ru
LEAD_REPLY_TO_EMAIL=manager@example.ru
```

### 2. Заполните каждую переменную

`RESEND_API_KEY`

- Это секретный API-ключ от сервиса Resend.
- Именно он используется сервером для отправки email.
- Получить ключ можно в панели управления Resend после создания аккаунта и настройки домена отправителя.
- Никогда не вставляйте этот ключ в `index.html`, `script.js` или любой клиентский код.

`LEAD_TO_EMAIL`

- Адрес, на который будут приходить заявки с сайта.
- Обычно это рабочая почта компании или менеджера.
- Пример: `sales@liftkomplekt.ru`

`LEAD_FROM_EMAIL`

- Адрес отправителя, от имени которого сервер будет отправлять письма.
- Этот адрес должен быть разрешён в вашем аккаунте Resend и обычно привязан к подтверждённому домену.
- Пример: `website@liftkomplekt.ru`

`LEAD_REPLY_TO_EMAIL`

- Необязательная переменная.
- Если указана, то при нажатии `Ответить` в почтовом клиенте ответ уйдёт на этот адрес.
- Если отдельный адрес для ответов не нужен, можно оставить тот же, что и в `LEAD_TO_EMAIL`, либо не использовать эту переменную совсем.

### 3. Пример для реального проекта

```env
RESEND_API_KEY=re_1234567890_example
LEAD_TO_EMAIL=info@liftkomplekt.ru
LEAD_FROM_EMAIL=noreply@liftkomplekt.ru
LEAD_REPLY_TO_EMAIL=info@liftkomplekt.ru
```

### 4. Важные правила

- Файл `.env` должен находиться в корне проекта.
- Не коммитьте `.env` в git.
- `.env.example` можно хранить в репозитории, потому что в нём только шаблон без реальных секретов.
- Все переменные должны задаваться без кавычек, если в значении нет особой необходимости.
- После изменения `.env` обычно нужно перезапустить локальный сервер, чтобы он перечитал переменные окружения.

### 5. Что будет, если переменные не заданы

Если не указать `RESEND_API_KEY`, `LEAD_TO_EMAIL` или `LEAD_FROM_EMAIL`, серверный endpoint не сможет отправить письмо. В этом случае форма вернёт пользователю сообщение на русском о временной недоступности сервиса.

### 6. Как проверить, что всё настроено правильно

1. Создайте `.env` по примеру из `.env.example`.
2. Укажите реальный `RESEND_API_KEY`.
3. Укажите реальный почтовый адрес получателя в `LEAD_TO_EMAIL`.
4. Укажите подтверждённый адрес отправителя в `LEAD_FROM_EMAIL`.
5. Запустите проект в окружении, где работает серверный endpoint.
6. Отправьте тестовую заявку через форму.
7. Проверьте, что письмо пришло на адрес из `LEAD_TO_EMAIL`.

### 7. Частые ошибки

- Использован тестовый или неверный API-ключ.
- В `LEAD_FROM_EMAIL` указан адрес, который не подтверждён в Resend.
- `.env` создан не в корне проекта.
- После изменения `.env` сервер не был перезапущен.
- Секрет случайно вставлен в клиентский JavaScript вместо серверной конфигурации.
